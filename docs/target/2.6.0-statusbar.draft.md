# 状态栏

> 底部状态信息区，显示运行时系统状态。只读展示，无用户交互。

## 显示项

| 项目 | 说明 | 来源 |
|------|------|------|
| AI 后端可用性 | Python 推理后端是否在线（连接中 / 已连接 / 断开） | 引擎事件 |
| GPU 可用性 | GPU 是否可用（可用 / 不可用），不可用时 GPU 节点禁用 | 引擎事件 |
| 执行进度 | 当前节点执行进度（节点名 + 百分比），空闲时隐藏 | 引擎事件 |
| CPU 占用 | 当前 CPU 使用率（%） | 系统采样 |
| 内存占用 | 已用 / 总物理内存 | 系统采样 |
| GPU 显存 | 已用 / 总显存（VRAM） | 系统采样 |
| GPU 利用率 | GPU 计算核心占用率（%） | 系统采样 |
| FPS | 界面渲染帧率（仅 debug 模式显示） | iced 运行时 |

## API

状态栏项通过注册机制声明，不硬编码。新增状态指示器只需注册一个 `StatusItem`：

```rust
statusbar.register(StatusItem {
    id:       "backend",
    position: Position::Left,
    priority: 10,
    render:   Box::new(|| match backend.status() {
        Connected    => StatusView::text("已连接").color(token("status.connected")),
        Connecting   => StatusView::text("连接中..."),
        Disconnected => StatusView::text("未连接").color(token("status.disconnected")),
    }),
});
```

`priority` 控制同侧排列顺序，`position` 区分左/右对齐。`render` 每帧求值，返回要显示的内容。

## 刷新机制

- **引擎事件驱动**：后端连接状态、执行进度由交互控制器接收引擎事件后推送刷新。
- **定时采样**：CPU、内存、GPU 指标由后台定时器按固定间隔（约 1s）采样，独立于引擎事件。

状态栏自身不持有业务状态，所有数据均为只读快照。

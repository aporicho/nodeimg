# 主题系统

> 全局视觉样式管理。所有组件的外观通过主题系统查询，不硬编码任何视觉值。

## 设计原则

- **单一来源**：所有颜色、尺寸、字体均从主题查询，无硬编码
- **热切换**：切换主题时所有组件立即更新，不需要重启，不缓存颜色值
- **iced 集成**：实现 iced 的 `Theme` trait，原生控件（按钮、滚动条等）自动跟随主题

## Token 架构

三层结构，组件只引用组件 token：

```
原始值（Primitive）
  #1A1A2E, #6DBFA0, 6px, 14px...
      ↓
语义 token（Semantic）
  bg.primary, accent, text.secondary...
      ↓
组件 token（Component）
  node.header.bg, pin.image.color, canvas.grid.color...
```

动态推导：衍生色（悬停、禁用）由基础值自动计算，不单独定义 token：
- 悬停 = 基础色 ± 10% 亮度
- 禁用 = 基础色 × 40% 透明度

### Token 注册 API

组件 token 支持动态注册，新增功能无需修改主题文件。`node.category.*` 由 `CategoryRegistry` 在分类注册时自动注入，不在主题文件里硬编码：

```rust
// CategoryRegistry 注册分类时，自动向主题注册对应 token
theme_registry.register_token(
    "node.category.data",
    TokenDef {
        light: Color::hex("#4A90D9"),
        dark:  Color::hex("#5BA3E8"),
    }
);
```

同理，自定义 DataType 注册时可注册对应的 `pin.*` token。

## 色彩基调（变更：从暗色优先改为白色极简风 shadcn zinc 色系）

基础色采用 shadcn zinc 色系，白色极简风为默认主题：

| 语义 | 亮色值 | 说明 |
|------|-------|------|
| `bg.primary` | `#fafafa` (zinc-50) | 主背景（画布背景） |
| `bg.secondary` | `#ffffff` | 面板/卡片背景 |
| `bg.overlay` | `#ffffff` | 弹窗/浮层背景 |
| `border` | `#e4e4e7` (zinc-200) | 通用边框 |
| `text.primary` | `#18181b` (zinc-900) | 主文字/强调色 |
| `text.secondary` | `#71717a` (zinc-500) | 次级文字 |
| `text.disabled` | `#a1a1aa` (zinc-400) | 禁用文字 |
| `accent` | `#18181b` (zinc-900) | 选中/强调（深色） |

## 模式

默认亮色（白色极简风），同时支持暗色模式，通过工具栏切换或跟随系统自动切换。用户选择持久化到 `AppConfig`。（变更：原暗色/亮色对等，现以亮色为主默认模式）

## 尺寸 / 间距系统

基于 4px 网格，所有间距为 4 的倍数：

| 级别 | 值 | 用途 |
|------|---|------|
| `space.1` | 4px | 控件内间距 |
| `space.2` | 8px | 面板内边距 |
| `space.3` | 12px | 控件间距 |
| `space.4` | 16px | 节点内边距 |
| `space.6` | 24px | 区块间距 |

## 无障碍

- **色盲模式**：引脚和执行状态在纯色之外增加形状/图案辅助区分
- **对比度校验**：切换主题时自动验证文字/背景对比度，不满足最低要求时警告

## 色彩空间说明

UI 颜色使用 sRGB，节点内图像预览使用线性色彩空间。两者不混用，主题系统只负责 sRGB UI 颜色。

# 预览面板

> 显示选中节点的输出预览。根据输出类型自动切换渲染模式。

## 输出类型与渲染模式

| 输出类型 | 渲染模式 | 说明 |
|---------|---------|------|
| Image / Mask | GPU 纹理直接渲染 | wgpu render pass，无 CPU 回读 |
| Video | 帧解码 → GPU 纹理渲染 | 解码线程上传帧，复用图像渲染路径 |
| Text | 文本显示区域 | iced 原生文本 widget，可滚动 |
| 无输出 / 不支持类型 | 空白占位 | — |

## 图像 / Mask 渲染

自建 `Shader` widget，在 `Primitive::prepare()` / `draw()` 回调里直接操作 wgpu render pass。引擎产出的 `GpuTexture` 留在显存，面板直接 sample 输出到屏幕，不经过 CPU。

```
引擎 GpuTexture（显存）
    ↓ sample（wgpu render pass）
预览面板输出（屏幕）
```

支持：
- 滚轮缩放 / 拖拽平移
- 像素级检查：悬停显示坐标 + RGBA 值
- 缩放比例显示

## 视频渲染

帧解码在后台线程进行（`video-rs` / `ffmpeg-next`），解码后的帧上传为 wgpu 纹理，复用图像渲染路径显示。

```
视频文件 / 视频帧数据
    ↓ 后台线程解码（video-rs）
    ↓ 上传 wgpu 纹理
    ↓ sample（wgpu render pass）
预览面板输出（屏幕）
```

播放控制：

| 控件 | 说明 |
|------|------|
| 播放 / 暂停 | 空格键或按钮 |
| 进度条 | 拖拽跳转到任意帧 |
| 帧率同步 | 按视频 FPS 上传新帧，iced 定时刷新 |

## 文本渲染

文本输出使用 iced 原生文本 widget，支持滚动查看。不走 GPU 渲染路径。

## API

预览面板通过注册机制支持新输出类型，不硬编码渲染逻辑：

```rust
preview.register_renderer(PreviewRenderer {
    data_type: DataType::Image,
    render:    Box::new(|output, area, wgpu_ctx| {
        // wgpu render pass
    }),
});
```

新增输出类型（如 Audio）只需注册对应 `PreviewRenderer`，面板根据当前节点输出的 `DataType` 自动选择渲染器。无匹配渲染器时显示空白占位。

## 刷新时机

交互控制器在以下情况通知预览面板刷新：
- 画布选中节点变化
- 引擎执行完成（节点输出更新）

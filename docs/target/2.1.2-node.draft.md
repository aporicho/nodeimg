# 节点

> 画布中单个节点的视觉结构和外观配色。参数控件系统详见 [2.1.3](./2.1.3-param-widgets.draft.md)。

## 节点外观建模（变更：统一模型，去掉 Content/Params 分区和折叠）

节点外观由 `NodeDef` 唯一确定，无需额外运行时信息。不存在"模式"区分，所有节点统一处理：

```
NodeDef {
    name, category,              → 标签（卡片外上方：小字节点名 + 分类色点）
    inputs, outputs,             → 引脚（⊕ 按钮 → 展开引脚列表）
    params,                      → 控件（DataType + Constraint 自动映射）
    inline_pins: Vec<String>,    → 哪些 pin 的预览控件内联显示在卡片内
    panel: Option<PanelHint>,    → 哪些 param 移到弹出面板里
}
```

卡片内的控件 = `inline_pins` 的预览控件 + `params` 的自动映射控件。视觉差异由控件组合自然产生。

### 节点卡片结构

```
  文本生成  ●                    ← 标签（卡片外上方，小字 + 分类色点）
┌──────────────────────────┐
│  [image preview]         │  ← inline_pins 预览控件（若有）
│  提示词  [............]  │  ← params 控件
│  模型    [GPT-4      ▼] │
│  温度    [----0.7----]  │
└──────────────────────────┘
```

- **标签**：节点名称在卡片外上方，小字体 + 分类色圆点，不占卡片空间
- **卡片（白色）**：白色背景、轻边框，无 header 色块。控件始终全部显示，无折叠（变更：原有 header 色块背景；原 params 区可折叠，改为始终展开）
- **引脚**：通过 ⊕ 按钮触发，不常驻显示于卡片边缘（见"引脚交互"一节）
- **弹出面板**（可选）：选中节点后在卡片底部弹出，承载不适合内联的参数（`PanelHint` 描述）

### 两种控件数据来源

| 来源 | 说明 | 例子 |
|------|------|------|
| param | 用户输入/调整的参数值 | Slider、Dropdown、TextArea |
| pin | 执行结果的预览 | ImagePreview（输出图片） |

## 引脚交互：两级展开 + Safe Triangle（变更：原为卡片边缘圆点，改为 ⊕ 按钮展开）

1. Hover 节点 → 两侧出现 ⊕ 按钮
2. Hover ⊕ → 展开引脚列表（彩色圆点 + 名称）
3. 鼠标从 ⊕ 移向列表时，Safe Triangle 算法保证列表不会闪退

引脚圆点颜色仍按数据类型配色，悬停兼容性高亮逻辑不变。

## 配色（变更：从暗色主题改为白色极简风 shadcn 风格）

### 节点卡片基础色

| 元素 | 值 | 说明 |
|------|---|------|
| 卡片背景 | `#ffffff` | 白色 |
| 卡片边框 | `#e4e4e7` | zinc-200 |
| 卡片阴影 | 轻投影 | 与背景区分 |
| 选中边框 | `#18181b` | zinc-900 深色 + ring 阴影 |
| 画布背景 | `#fafafa` | zinc-50，圆点网格 24px 间距 |

### 分类配色（标签色点）（变更：原用于 header 背景，改为卡片外标签的小色点）

分类通过 `CategoryRegistry` 注册，颜色随分类一起声明，新增分类无需修改主题文件：

```rust
category_registry.register(CategoryDef {
    id:            "data",
    label:         "输入 / 输出",
    default_color: Color::hex("#4A90D9"),
});
```

内置分类（供参考，非硬编码）：

| 分类 | 色调 | 代表节点 |
|------|------|---------|
| `data` | 蓝 | LoadImage、SaveImage |
| `generate` | 紫 | 纯色、渐变、噪声 |
| `color` | 橙 | 亮度、曲线、色调 |
| `transform` | 绿 | 缩放、裁剪、旋转 |
| `filter` | 粉 | 模糊、锐化、降噪 |
| `composite` | 金 | 混合、蒙版 |
| `tool` | 灰 | 预览、调试工具 |
| `ai` | 紫红 | 采样、模型加载 |

### 数据类型配色（引脚）

引脚圆点按数据类型着色，用户通过颜色判断连线兼容性：

| 数据类型 | 色调 |
|---------|------|
| `image` | 蓝 |
| `mask` | 青绿 |
| `float` | 橙 |
| `int` | 青 |
| `string` | 灰 |
| `color` | 紫 |
| `bool` | 红 |
| `model` | 亮紫 |
| `clip` | 金 |
| `vae` | 红 |
| `conditioning` | 淡紫 |
| `latent` | 粉 |

## 布局常量

| 常量 | 值 | 说明 |
|------|---|------|
| 节点圆角 | 6px | 节点体和头部的圆角半径 |
| 节点最小宽度 | 180px | 节点在画布上的最小宽度 |
| 引脚半径 | 5px | 输入/输出引脚圆点半径 |
| 面板内边距 | 8px | 节点内边距 |
| 控件间距 | 4px | 参数控件之间的垂直间距 |


## 执行状态显示（变更：无 header 色块，状态通过卡片边框/标签区域体现）

节点根据 `UIState.execution` 中的状态显示视觉反馈：

| 状态 | 显示 |
|------|------|
| 等待执行 | 无变化 |
| 执行中 | 卡片顶部显示进度条或旋转指示器 |
| 执行完成 | 恢复正常 |
| 执行出错 | 卡片边框变红，标签区显示错误图标 |

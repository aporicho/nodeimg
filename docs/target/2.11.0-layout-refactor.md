# 布局引擎重构：消除中间 LayoutBox 转换

> 状态：已完成

## 概述

消除了布局流程中多余的 `LayoutBox` 临时树。Flexbox 引擎通过 `LayoutTree` trait 直接操作 PanelTree，叶子节点下沉到渲染图元（Text），Widget trait 删除。

## 最终实现

### LayoutTree trait

```rust
// widget/layout/types.rs
pub trait LayoutTree {
    type NodeId: Copy;
    fn style(&self, node: Self::NodeId) -> &BoxStyle;
    fn children(&self, node: Self::NodeId) -> Vec<Self::NodeId>;
    fn set_rect(&mut self, node: Self::NodeId, rect: Rect);
    fn scroll_offset(&self, node: Self::NodeId) -> f32;
    fn set_content_height(&mut self, node: Self::NodeId, height: f32);
}
```

PanelTree 直接实现此 trait，无需 Adapter 包装。

### 叶子 = 渲染图元

Button 从 Widget 分解为 Container + Text 叶子：
```rust
Desc::Container { id, style, decoration, children: vec![
    Desc::Leaf { id, style, kind: LeafKind::Text { content, font_size, color } }
]}
```

文字度量在构建 Desc 时完成（`measure_text`），存为 `Size::Fixed`。布局阶段不需要 Renderer。

### 删除的类型

- `LayoutBox` — 中间临时树节点
- `LayoutResult` — 中间结果容器
- `ScrollArea` — 滚动状态移至 PanelNode
- `Widget` trait — 控件改为返回 Desc 的函数

### 新增的类型

- `LayoutTree` trait — 布局引擎的树抽象
- `LeafKind` — 渲染图元（Text）
- `Decoration` — 视觉装饰（背景/边框/圆角）

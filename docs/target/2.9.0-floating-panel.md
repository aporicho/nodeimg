# 通用浮动面板

> 浮在画布上方的容器组件。提供统一的标题栏、内容区、resize handle，管理拖拽移动和尺寸调整。

## 视觉结构

```
┌─────────────────────────┐
│ Title              28px │ ← 标题栏（拖拽移动触发区）
├─────────────────────────┤
│                         │
│       content           │ ← 内容区（由使用者填充）
│                         │
│                       ◢ │ ← resize handle（16x16）
└─────────────────────────┘
```

外框：白底、1px 边框、圆角 8px、阴影。

## 定位模型

面板不存绝对位置，只存**相对于默认布局位置的偏移 `offset: Vector`**。

默认位置由 `lib.rs` 中的 iced 布局决定（`center`、`align_x(Right)` 等），`FloatingPanel` 通过 `float().translate(offset)` 叠加偏移。这样面板会跟随窗口缩放自动调整基准位置，拖拽只改变偏移量。

## 全屏捕获层

iced 的 `mouse_area` 给出的坐标是**相对于自身 widget 的局部坐标**。如果直接在面板上捕获拖拽的 `on_move`，面板移动后坐标基准变了，导致跳跃。

解决方案：面板只发 `DragStart` / `ResizeStart`，后续的 `Move` 和 `End` 事件由 `lib.rs` 中一个全屏透明 `mouse_area`（覆盖整个窗口）统一捕获，保证所有坐标都在窗口坐标系下。

```
stack![
    canvas,           // 底层：节点画布
    preview,          // 浮动面板
    toolbar,          // 浮动面板
    drag_layer,       // 全屏捕获层（仅交互时出现）
]
```

`drag_layer` 只在某个面板处于 `is_dragging()` 或 `is_resizing()` 状态时才加入 stack。

## 状态

| 字段 | 说明 |
|------|------|
| `offset: Vector` | 相对默认位置的偏移 |
| `size: Size` | 面板宽高（最小 120x80） |
| `dragging: bool` | 拖拽中 |
| `resizing: bool` | resize 中 |

drag 和 resize 互斥，同时只有一个激活。

## 使用方式

```rust
// 1. 面板持有 FloatingPanel
pub struct MyPanel {
    pub panel: FloatingPanel,
}

// 2. Message 包含 panel::Event
pub enum Message {
    Panel(panel::Event),
    // ...自己的消息
}

// 3. update 转发事件
Message::Panel(event) => self.panel.handle_event(event),

// 4. view 调用通用框架
pub fn view(&self) -> Element<'_, Message> {
    let content = /* 你的内容 widget */;
    self.panel.view("Title", content, Message::Panel)
}

// 5. lib.rs 中用 float + translate 定位，注册到全屏捕获层
```

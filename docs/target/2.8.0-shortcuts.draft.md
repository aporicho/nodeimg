# 快捷键系统

> 全局快捷键注册和路由。核心目标：任何操作都能以最小代价注册快捷键，无需修改中心文件。

## API 设计

### 注册

每个组件在初始化时声明式注册自己的快捷键：

```rust
shortcut_registry.register(KeyBinding {
    key:         Key::Cmd(Z),
    context:     Context::Global,
    action:      Action::Undo,
    description: "撤销",
});
```

注册即生效，无需修改任何中心路由表。组件销毁时自动注销。

### Context（上下文）

快捷键绑定在特定上下文下生效，更具体的上下文优先于更宽泛的：

```
Global              — 任意焦点
  Canvas            — 画布获得焦点
    NodeEditing     — 节点内输入控件获得焦点
  Preview           — 预览面板获得焦点
    VideoPlaying    — 视频节点播放中
  Chat              — AI 对话面板获得焦点
```

同一按键在不同 Context 下可绑定不同操作，由当前焦点决定路由。

### Action

Action 是具名的操作描述，不是匿名函数：

```rust
enum Action {
    Undo,
    Redo,
    Save,
    DeleteSelected,
    // ...自定义 Action
}
```

快捷键系统只负责路由，Action 的实际处理由对应控制器执行。新增操作 = 新增 Action 变体 + 注册 KeyBinding，无需改路由逻辑。

### 冲突处理

- 注册时检测冲突，开发模式下 panic，生产模式下日志警告
- 相同 Context + 相同按键 = 冲突，不允许覆盖

### 查询

快捷键系统暴露完整注册表，供帮助面板、快捷键提示 UI 查询：

```rust
registry.all_bindings()                    // 全部快捷键
registry.bindings_for(Context::Canvas)     // 特定上下文
registry.key_for(Action::Undo)             // 查某操作的按键
```

## 内置快捷键

以下为默认注册的快捷键，仅供参考，不是固定设计：

| 上下文 | 快捷键 | 操作 |
|--------|--------|------|
| Global | Cmd+S | 保存 |
| Global | Cmd+Z | 撤销 |
| Global | Cmd+Shift+Z | 重做 |
| Global | Cmd+N | 新建 |
| Global | Cmd+O | 打开 |
| Canvas | Cmd+C/V/X | 复制/粘贴/剪切 |
| Canvas | Cmd+A | 全选 |
| Canvas | Delete | 删除选中 |
| Canvas | Cmd+D | 复制并偏移 |
| Canvas | F | 适应视口 |
| Canvas | 方向键 | 微移节点 |
| Canvas | Escape | 取消当前交互 |
| Preview | F | 适应窗口 |
| Preview | 1 | 100% 缩放 |
| VideoPlaying | Space | 播放/暂停 |

# 引擎共享契约

> GUI、CLI、AI 操作员共用的一套引擎边界。第一版以同进程函数调用 + 广播事件为前提，不定义网络协议。

---

## 1. 目标与边界

本文件定义三类前端与引擎之间的唯一共享 contract：

- GUI -> 引擎
- CLI -> 引擎
- AI 操作员 -> 引擎

本文件纳入的能力范围：

- Project 生命周期
- Graph 编辑
- Execution 执行
- Artifact 产物
- Backend 可用性
- Event 事件订阅

本文件明确不纳入的内容：

- `Selection`
- `Viewport`
- `PanelLayout`
- `scroll_to_node`
- Toast / Modal / Command Palette
- AI 对话历史
- 低层进程控制（如 `start_backend` / `stop_backend`）

这些能力属于 GUI / App 层 contract，不属于三端共享引擎边界。

---

## 2. 调用模型

### 2.1 总体模型

- **命令（Command）**：会修改引擎状态；成功后可触发事件
- **查询（Query）**：只读快照；不修改状态，不触发事件
- **事件（Event）**：引擎通过广播流异步推送给订阅者

### 2.2 交互方式

- **同步命令**：图编辑、项目切换等操作在状态提交后返回
- **异步命令**：`execute`、`ensure_backend`、`reconnect_backend` 在“请求已接受”后返回，后续进度通过事件通知
- **查询**：返回调用时刻的一致性快照
- **事件订阅**：每个调用方获得独立事件流；只接收订阅之后的新事件，不回放历史

### 2.3 非目标

本文件不定义：

- HTTP / IPC / MCP wire format
- 多进程对象传输格式
- GUI 控件层 action

如果未来需要远程调用，引擎可以在本 contract 之上再包一层协议适配器。

## 3. Facade 轮廓

第一版共享边界建议收敛为单一 `EngineFacade`：

```rust
trait EngineFacade { // 三端共享的统一引擎入口
    // Project
    fn get_project_state(&self) -> ProjectState; // 读取当前项目状态
    fn new_project(&self, req: NewProjectRequest) -> Result<ProjectState, EngineError>; // 新建内存中的 untitled 项目
    fn open_project(&self, req: OpenProjectRequest) -> Result<ProjectState, EngineError>; // 打开磁盘上的 .nodeimg 项目
    fn save_project(&self) -> Result<ProjectState, EngineError>; // 保存到当前项目路径
    fn save_project_as(&self, req: SaveProjectAsRequest) -> Result<ProjectState, EngineError>; // 另存为新路径
    fn close_project(&self, req: CloseProjectRequest) -> Result<ProjectState, EngineError>; // 关闭当前项目

    // Graph
    fn get_graph_snapshot(&self) -> Result<Graph, EngineError>; // 读取当前图快照
    fn list_node_defs(&self) -> Vec<NodeDef>; // 列出全部已注册节点定义
    fn get_node_def(&self, type_id: &str) -> Option<NodeDef>; // 查询某个节点类型定义
    fn add_node(&self, req: AddNodeRequest) -> Result<NodeId, EngineError>; // 添加节点并返回新节点 id
    fn remove_node(&self, node_id: NodeId) -> Result<(), EngineError>; // 删除节点及其相关连线
    fn connect(&self, req: ConnectRequest) -> Result<(), EngineError>; // 创建一条连线
    fn disconnect(&self, req: DisconnectRequest) -> Result<(), EngineError>; // 删除一条连线
    fn set_param(&self, req: SetParamRequest) -> Result<(), EngineError>; // 修改节点参数
    fn move_node(&self, req: MoveNodeRequest) -> Result<(), EngineError>; // 修改节点位置
    fn apply_batch(&self, req: EditBatchRequest) -> Result<EditBatchResult, EngineError>; // 原子提交一组图编辑
    fn undo(&self) -> Result<(), EngineError>; // 撤销上一次提交
    fn redo(&self) -> Result<(), EngineError>; // 重做上一次撤销
    fn replace_graph(&self, graph: Graph) -> Result<(), EngineError>; // 用一整张图替换当前图
    fn clear_cache(&self) -> Result<(), EngineError>; // 清空执行缓存，不改图
    fn validate_graph(&self) -> Result<ValidationReport, EngineError>; // 校验当前图是否存在结构或参数问题
    fn upstream(&self, node_id: NodeId) -> Result<Vec<NodeId>, EngineError>; // 查询某个节点的全部上游节点
    fn downstream(&self, node_id: NodeId) -> Result<Vec<NodeId>, EngineError>; // 查询某个节点的全部下游节点
    fn resolve_subgraph(&self, target: ExecuteTarget) -> Result<SubgraphInfo, EngineError>; // 解析某次执行会涉及的子图和拓扑顺序

    // Execution
    fn get_execution_state(&self) -> ExecutionState; // 读取当前执行状态
    fn get_execution_result(&self, execution_id: ExecutionId) -> Result<Option<ExecutionResultSummary>, EngineError>; // 读取某次执行的终态摘要
    fn execute(&self, req: ExecuteRequest) -> Result<ExecutionTicket, EngineError>; // 发起一次执行
    fn cancel_execution(&self) -> Result<(), EngineError>; // 请求取消当前执行

    // Artifact
    fn get_artifact_history(&self, node_id: NodeId) -> Result<Vec<ArtifactRecord>, EngineError>; // 读取某个节点的历史产物
    fn select_artifact(&self, req: SelectArtifactRequest) -> Result<(), EngineError>; // 选择某个历史产物作为当前版本

    // Backend
    fn get_backend_status(&self) -> BackendStatus; // 读取后端当前状态
    fn ensure_backend(&self) -> Result<BackendStatus, EngineError>; // 确保后端处于可用状态
    fn reconnect_backend(&self) -> Result<BackendStatus, EngineError>; // 主动要求重新连接后端

    // Events
    fn subscribe_events(&self) -> EventStream<EngineEvent>; // 订阅后续引擎事件流
}
```

说明：

- `EventStream<EngineEvent>` 代表任意“可独立消费的广播订阅句柄”；具体实现可为 `broadcast::Receiver<EngineEvent>` 或等价物
- `Graph`、`NodeDef`、`Value` 等共享模型在 `0.1.0-models.md` 中展开；本文件先定义边界上的最小必需字段和语义

## 4. Shared Types

### 4.1 Project

```rust
type ProjectPath = PathBuf; // 指向 .nodeimg bundle 的磁盘路径

struct ProjectState { // 当前项目的整体状态
    path: Option<ProjectPath>, // 当前项目路径；untitled 项目时为空
    display_name: String, // 给 GUI / CLI 展示的项目名称
    dirty: bool, // 是否存在未保存修改
    is_open: bool, // 当前是否有项目处于打开状态
    graph_version: u64, // 当前图版本号，用于前端判断是否变化
}

struct NewProjectRequest { // 新建项目请求
    discard_unsaved: bool, // 遇到脏项目时，是否允许直接丢弃未保存修改
}

struct OpenProjectRequest { // 打开项目请求
    path: ProjectPath, // 要打开的 .nodeimg 路径
    discard_unsaved: bool, // 遇到脏项目时，是否允许直接丢弃未保存修改
}

struct SaveProjectAsRequest { // 另存为请求
    path: ProjectPath, // 目标保存路径
}

struct CloseProjectRequest { // 关闭项目请求
    discard_unsaved: bool, // 遇到脏项目时，是否允许直接丢弃未保存修改
}
```

语义：

- `new_project` 创建的是**内存中的 untitled 项目**，`path = None`
- `save_project` 对 `path = None` 的项目返回 `PathRequired`
- `save_project_as` 成功后将当前项目路径切换为新路径

### 4.3 Batch Edit

`apply_batch` 用于把多步图编辑原子提交为单个 undo 单元，主要服务于：

- AI 操作员的一次多步操作
- GUI 粘贴 / 批量生成 / 多节点重排
- CLI 的声明式图编辑

```rust
enum BatchNodeRef { // batch 里引用节点的方式
    Existing(NodeId), // 引用已经存在的节点
    Created(String), // 引用本批次刚新建的节点，值是客户端提供的临时 key
}

struct BatchPinRef { // batch 内用的引脚引用
    node: BatchNodeRef, // 引脚所属节点，可是旧节点也可是本批次新节点
    pin: String, // 引脚名称
}

enum GraphEdit { // batch 中的一条图编辑指令
    AddNode { // 新建一个节点
        client_key: String, // 这个新节点在 batch 内的临时名字
        type_id: String, // 节点类型 id
        position: Vec2, // 新节点位置
        param_overrides: HashMap<String, Value>, // 初始参数覆盖值
    },
    RemoveNode { // 删除一个已有节点
        node_id: NodeId, // 要删除的节点 id
    },
    Connect { // 建立一条连线
        from: BatchPinRef, // 源引脚
        to: BatchPinRef, // 目标引脚
    },
    Disconnect { // 删除一条连线
        from: BatchPinRef, // 源引脚
        to: BatchPinRef, // 目标引脚
    },
    SetParam { // 修改一个参数
        node: BatchNodeRef, // 被修改参数的节点
        param: String, // 参数名
        value: Value, // 参数值
        preview: bool, // 是否仅作为预览修改
    },
    MoveNode { // 移动一个节点
        node: BatchNodeRef, // 被移动的节点
        position: Vec2, // 新位置
    },
}

struct EditBatchRequest { // 一次批量编辑请求
    edits: Vec<GraphEdit>, // 这次要按顺序执行的所有编辑
    label: Option<String>, // 可选的人类可读标签，便于 undo 历史展示
}

struct EditBatchResult { // 一次批量编辑成功后的返回值
    created_nodes: HashMap<String, NodeId>, // 临时 key 到真实 NodeId 的映射
    graph_version: u64, // 提交完成后的图版本号
}
```

约束：

- batch 内编辑按顺序执行
- 任一步失败，整个 batch 回滚，不产生部分提交
- 整个 batch 只产生一个 `GraphChanged` 事件和一个 undo 单元

### 4.4 Topology & Validation

```rust
struct ValidationReport { // 一次图校验的完整结果
    is_valid: bool, // 当前图是否通过全部校验
    issues: Vec<ValidationIssue>, // 所有发现的问题列表
}

struct ValidationIssue { // 单个校验问题
    code: ValidationIssueCode, // 问题类别
    node_id: Option<NodeId>, // 如果能定位到具体节点，则带上节点 id
    message: String, // 给人看的错误说明
}

enum ValidationIssueCode { // 共享 contract 层面的稳定问题分类
    CycleDetected, // 图中出现环
    TypeMismatch, // 连线两端类型不兼容
    MissingNode, // 连线或引用指向了不存在的节点
    MissingNodeType, // 节点引用了未注册的节点类型
    InvalidParamValue, // 参数值不满足约束
}

struct SubgraphInfo { // 一次执行目标解析出来的子图信息
    target: ExecuteTarget, // 这次解析的执行目标
    nodes: Vec<NodeId>, // 被包含在子图中的全部节点
    order: Vec<NodeId>, // 按拓扑顺序排列的执行顺序
}
```

语义：

- `validate_graph` 是纯查询，不修改图，也不触发事件
- `upstream` / `downstream` 返回的是传递闭包，而不是只返回一层邻居
- `resolve_subgraph` 返回“如果现在执行，会涉及哪些节点、按什么顺序执行”的只读结果

### 4.5 Execution

```rust
type ExecutionId = u64; // 一次执行的唯一编号

struct ExecutionTicket { // execute 成功受理后的凭证
    execution_id: ExecutionId, // 这次执行的 id
}

enum ExecuteTarget { // 执行目标
    Graph, // 执行整张当前图
    Node(NodeId), // 执行某个节点及其全部上游依赖
}

enum ExecutionMode { // 执行模式
    Manual, // 用户明确触发的完整执行
    AutoPreview, // 面向实时预览的低副作用执行
}

struct ExecuteRequest { // 执行请求
    target: ExecuteTarget, // 这次要执行哪一部分图
    mode: ExecutionMode, // 这次执行采用什么模式
}

struct ExecutionState { // 当前执行状态快照
    status: ExecutionStatus, // 当前是否空闲、执行中、取消中
    current: Option<RunningExecution>, // 如果有执行在跑，这里给出它的概要信息
}

enum ExecutionStatus { // 执行状态枚举
    Idle, // 当前没有执行
    Running, // 当前有执行在运行
    Cancelling, // 已收到取消请求，正在收尾
}

struct RunningExecution { // 正在运行中的执行概要
    execution_id: ExecutionId, // 这次运行的 id
    target: ExecuteTarget, // 这次运行的目标
    mode: ExecutionMode, // 这次运行的模式
}

struct ExecutionResultSummary { // 一次执行结束后的摘要
    execution_id: ExecutionId, // 这次执行的 id
    status: ExecutionTerminalStatus, // 最终是成功、取消还是错误结束
    target: ExecuteTarget, // 这次执行的目标
    mode: ExecutionMode, // 这次执行的模式
    finished_nodes: Vec<NodeId>, // 成功完成的节点集合
    failed_nodes: Vec<NodeId>, // 失败的节点集合
}

enum ExecutionTerminalStatus { // 一次执行的终态
    Finished, // 正常完成
    Cancelled, // 被取消后结束
    Error, // 因错误终止
}
```

语义：

- `ExecuteTarget::Graph` 表示执行当前图的完整可执行部分
- `ExecuteTarget::Node(NodeId)` 表示执行该节点及其全部上游依赖
- `ExecutionMode::Manual` 允许完整执行，包括 AI / API 节点和产物写入
- `ExecutionMode::AutoPreview` 面向本地实时预览；远程或昂贵执行器可选择跳过，不产生外部副作用
- `get_execution_result(execution_id)` 在执行未结束时返回 `Ok(None)`
- 引擎至少保留最近一次终态执行的 `ExecutionResultSummary`，更长的保留策略由实现决定

### 4.7 Backend

```rust
enum BackendStatus { // 后端当前可用性状态
    Disabled, // 当前配置下不使用后端
    Starting, // 正在启动或初始化连接
    Ready, // 已经可用，可以接收请求
    Disconnected, // 本来应该可用，但当前连接断开
    Error { message: String }, // 发生错误，并附带可展示消息
}
```

语义：

- `ensure_backend` 是幂等操作；目标是“保证当前可用”
- `reconnect_backend` 是显式恢复动作；目标是“丢失连接后重新建立可用状态”
- 本 contract 不暴露 `start_backend` / `stop_backend`，避免把本地进程管理细节固化到共享边界

## 5. Commands

### 5.1 Project Commands

| 接口 | 输入 | 返回 | 异步 | 事件 | undo | 说明 |
|------|------|------|------|------|------|------|
| `new_project` | `NewProjectRequest` | `ProjectState` | 否 | `ProjectChanged(New)`、`GraphChanged(GraphReplaced)` | 否 | 创建内存中的 untitled 空项目 |
| `open_project` | `OpenProjectRequest` | `ProjectState` | 否 | `ProjectChanged(Opened)`、`GraphChanged(GraphReplaced)` | 否 | 加载 `.nodeimg`，替换当前图和 artifact |
| `save_project` | - | `ProjectState` | 否 | `ProjectChanged(Saved)` | 否 | 保存到当前路径；untitled 项目返回 `PathRequired` |
| `save_project_as` | `SaveProjectAsRequest` | `ProjectState` | 否 | `ProjectChanged(SavedAs)` | 否 | 写入新路径并切换当前项目路径 |
| `close_project` | `CloseProjectRequest` | `ProjectState` | 否 | `ProjectChanged(Closed)`、`GraphChanged(GraphReplaced)` | 否 | 关闭当前项目，进入无项目状态 |

### 5.2 Graph Commands

| 接口 | 输入 | 返回 | 异步 | 事件 | undo | 说明 |
|------|------|------|------|------|------|------|
| `add_node` | `AddNodeRequest` | `NodeId` | 否 | `GraphChanged(NodeAdded)` | 是 | 使用 `NodeDef` 默认参数 + 覆盖参数创建节点 |
| `remove_node` | `NodeId` | `()` | 否 | `GraphChanged(NodeRemoved)` | 是 | 级联删除相关连线 |
| `connect` | `ConnectRequest` | `()` | 否 | `GraphChanged(Connected)` | 是 | 含单输入替换、类型兼容和环检测 |
| `disconnect` | `DisconnectRequest` | `()` | 否 | `GraphChanged(Disconnected)` | 是 | 删除指定连线 |
| `set_param` | `SetParamRequest` | `()` | 否 | `GraphChanged(ParamChanged)` | `preview=false` 时是；`preview=true` 时否 | 拖动中实时预览和最终提交共用同一接口 |
| `move_node` | `MoveNodeRequest` | `()` | 否 | `GraphChanged(NodeMoved)` | 否 | 默认不进入 undo 栈 |
| `apply_batch` | `EditBatchRequest` | `EditBatchResult` | 否 | `GraphChanged(BatchApplied)` | 是，一个 undo 单元 | 原子提交多步图编辑 |
| `undo` | - | `()` | 否 | `GraphChanged(Undo)` | 否 | 回退上一提交版本 |
| `redo` | - | `()` | 否 | `GraphChanged(Redo)` | 否 | 前进到下一提交版本 |
| `replace_graph` | `Graph` | `()` | 否 | `GraphChanged(GraphReplaced)` | 否 | 用外部图整体替换当前图，重置 undo/redo |
| `clear_cache` | - | `()` | 否 | 无 | 否 | 清理结果缓存和纹理缓存，不改图 |

### 5.3 Execution Commands

| 接口 | 输入 | 返回 | 异步 | 事件 | undo | 说明 |
|------|------|------|------|------|------|------|
| `execute` | `ExecuteRequest` | `ExecutionTicket` | 是 | `NodeStarted` / `NodeProgress` / `NodeFinished` / `NodeError` / `ExecutionFinished` / `ExecutionCancelled` / `ExecutionError` | 否 | 单飞行；已有执行时返回 `Busy` |
| `cancel_execution` | - | `()` | 是 | `ExecutionCancelled` 或后续完成事件 | 否 | 尽力取消，不回滚已完成副作用 |

### 5.4 Artifact Commands

| 接口 | 输入 | 返回 | 异步 | 事件 | undo | 说明 |
|------|------|------|------|------|------|------|
| `select_artifact` | `SelectArtifactRequest` | `()` | 否 | `ArtifactSelected` | 否 | 将历史产物切换为当前选用版本 |

### 5.5 Backend Commands

| 接口 | 输入 | 返回 | 异步 | 事件 | undo | 说明 |
|------|------|------|------|------|------|------|
| `ensure_backend` | - | `BackendStatus` | 是 | `BackendStatusChanged` | 否 | 保证当前可用；已可用时应快速返回 |
| `reconnect_backend` | - | `BackendStatus` | 是 | `BackendStatusChanged` | 否 | 主动恢复连接 |

## 6. Queries

| 接口 | 返回 | 说明 |
|------|------|------|
| `get_project_state` | `ProjectState` | 返回当前项目状态；无项目时 `is_open=false` |
| `get_graph_snapshot` | `Graph` | 返回当前图的只读快照；无项目时返回 `ProjectNotOpen` |
| `validate_graph` | `ValidationReport` | 返回当前图的结构和参数校验结果 |
| `upstream` | `Vec<NodeId>` | 返回某个节点的全部上游节点 |
| `downstream` | `Vec<NodeId>` | 返回某个节点的全部下游节点 |
| `resolve_subgraph` | `SubgraphInfo` | 返回某个执行目标会涉及的子图和拓扑顺序 |
| `list_node_defs` | `Vec<NodeDef>` | 返回所有已注册节点定义 |
| `get_node_def` | `Option<NodeDef>` | 返回指定节点类型定义 |
| `get_execution_state` | `ExecutionState` | 返回当前执行状态 |
| `get_execution_result` | `Option<ExecutionResultSummary>` | 返回指定执行 id 的终态摘要；未结束时为 `None` |
| `get_artifact_history` | `Vec<ArtifactRecord>` | 返回指定节点的产物历史 |
| `get_backend_status` | `BackendStatus` | 返回当前后端状态 |

### 7.2 Event Payload

```rust
struct ProjectChanged { // 项目变化事件
    reason: ProjectChangeReason, // 变化原因
    state: ProjectState, // 变化后的项目状态
}

enum ProjectChangeReason { // 项目变化原因枚举
    New, // 新建了项目
    Opened, // 打开了现有项目
    Saved, // 保存到当前路径
    SavedAs, // 另存为了新路径
    Closed, // 关闭了项目
}

struct GraphChanged { // 图变化事件
    reason: GraphChangeReason, // 变化原因
    graph_version: u64, // 变化后的图版本号
    dirty: bool, // 变化后项目是否变脏
}

enum GraphChangeReason { // 图变化原因枚举
    NodeAdded { node_id: NodeId }, // 新增了一个节点
    NodeRemoved { node_id: NodeId }, // 删除了一个节点
    Connected { from: PinRef, to: PinRef }, // 新建了一条连线
    Disconnected { from: PinRef, to: PinRef }, // 删除了一条连线
    ParamChanged { node_id: NodeId, param: String, preview: bool }, // 某个参数被修改
    NodeMoved { node_id: NodeId }, // 某个节点位置变化
    BatchApplied, // 一整个 batch 被提交
    Undo, // 执行了一次撤销
    Redo, // 执行了一次重做
    GraphReplaced, // 整张图被整体替换
}

struct NodeStarted { // 节点开始执行事件
    execution_id: ExecutionId, // 所属执行 id
    node_id: NodeId, // 开始执行的节点
}

struct NodeProgress { // 节点进度事件
    execution_id: ExecutionId, // 所属执行 id
    node_id: NodeId, // 产生进度的节点
    current: u32, // 当前进度值
    total: u32, // 总进度值
    preview_handle: Option<Handle>, // 可选的中间预览图句柄
}

struct NodeFinished { // 节点执行完成事件
    execution_id: ExecutionId, // 所属执行 id
    node_id: NodeId, // 已完成的节点
}

struct NodeError { // 节点执行失败事件
    execution_id: ExecutionId, // 所属执行 id
    node_id: NodeId, // 失败的节点
    error: EngineError, // 失败原因
}

struct ExecutionFinished { // 整轮执行成功结束事件
    result: ExecutionResultSummary, // 这轮执行的终态摘要
}

struct ExecutionCancelled { // 整轮执行取消事件
    result: ExecutionResultSummary, // 这轮执行取消后的终态摘要
}

struct ExecutionError { // 整轮执行错误事件
    result: ExecutionResultSummary, // 这轮执行错误结束时的终态摘要
    error: EngineError, // 整轮执行级别的错误
}

struct ArtifactSelected { // 历史产物被选中事件
    node_id: NodeId, // 所属节点
    artifact_id: ArtifactId, // 被选中的产物 id
}

struct ArtifactWriteFailed { // 产物写入失败事件
    node_id: NodeId, // 所属节点
    artifact_id: Option<ArtifactId>, // 如果已经分配 id，则带上产物 id
    error: EngineError, // 写入失败原因
}

struct BackendStatusChanged { // 后端状态变化事件
    status: BackendStatus, // 变化后的后端状态
}
```

### 7.3 事件顺序

- 同一 producer 发出的事件保持 FIFO 顺序
- 不同 producer 之间不承诺全局总顺序
- 单次执行内部，`NodeStarted -> NodeProgress* -> NodeFinished/NodeError` 对同一节点必须有序
- `execute` 成功返回后，订阅者最终必须收到以下之一：
  - `ExecutionFinished`
  - `ExecutionCancelled`
  - `ExecutionError`

## 8. Error Model

第一版错误先按稳定分类定义；具体错误消息由实现补充：

```rust
enum EngineError { // 共享边界上的稳定错误分类
    Busy, // 当前资源正忙，比如已有执行在跑
    ProjectNotOpen, // 当前没有打开项目，但调用方要求有项目
    ProjectDirty, // 当前项目有未保存修改，但调用方未允许丢弃
    PathRequired, // 需要路径，但当前没有提供或当前项目还是 untitled
    PathInvalid, // 路径本身不合法或不满足格式要求
    NodeTypeNotFound { type_id: String }, // 节点类型不存在
    NodeNotFound { node_id: NodeId }, // 节点实例不存在
    ParamNotFound { node_id: NodeId, param: String }, // 参数不存在
    InvalidValue { node_id: NodeId, param: String }, // 参数值非法
    InvalidConnection { reason: String }, // 连线非法，具体原因写在 reason 里
    ArtifactNotFound { node_id: NodeId, artifact_id: ArtifactId }, // 指定历史产物不存在
    BackendUnavailable, // 当前无法满足后端可用性要求
    ExecutionNotRunning, // 请求取消时其实没有执行在跑
    Io { message: String }, // 文件或系统 I/O 错误
    Internal { message: String }, // 未归类的内部错误
}
```

设计约束：

- `Busy` 主要用于单飞行执行冲突，以及执行中禁止的 whole-project 操作
- `ProjectDirty` 用于调用方未显式允许丢弃未保存改动时的项目切换/关闭请求
- `InvalidConnection` 的 `reason` 至少覆盖：环检测失败、类型不兼容、自连接、节点不存在
- `BackendUnavailable` 不等于 `Error` 状态；它表示调用时无法满足后端可用性要求

## 10. 语义规则

### 10.1 Project 规则

- 引擎允许“无项目”状态；此时 `get_project_state().is_open == false`
- `new_project` / `open_project` / `close_project` 若遇未保存改动且请求未允许丢弃，返回 `ProjectDirty`
- `new_project` / `open_project` / `close_project` 会重置：
  - undo / redo 栈
  - 当前执行状态
  - cache
  - artifact 选择状态

### 10.2 Graph 规则

- Graph 编辑命令在返回前完成状态提交
- `set_param(preview=true)` 不进入 undo 栈
- `move_node` 默认不进入 undo 栈
- `replace_graph` 不进入 undo 栈，并重置 undo / redo
- 执行中的图编辑是允许的；运行中的执行读取的是自己的快照，不受后续编辑影响
- `validate_graph`、`upstream`、`downstream`、`resolve_subgraph` 全部是纯查询，不产生状态变化和事件

### 10.3 Execution 规则

- 引擎采用 **single-flight**：同一时刻最多一个活动执行
- `execute` 在已有活动执行时返回 `Busy`
- `cancel_execution` 是尽力取消：
  - 已完成节点的结果不回滚
  - 已发生的远端副作用不回滚
- `AutoPreview` 是“低副作用预览模式”，远端和昂贵执行器可跳过
- 一次执行进入终态后，必须满足以下两个条件：
  - 发出一个终态事件：`ExecutionFinished` / `ExecutionCancelled` / `ExecutionError`
  - 可通过 `get_execution_result(execution_id)` 读取终态摘要

### 10.4 Batch 规则

- `apply_batch` 是原子提交
- batch 内新增节点通过 `client_key` 暴露给后续编辑引用
- 成功时返回 `created_nodes` 映射，便于调用方把临时 key 解析为真实 `NodeId`
- GUI 的粘贴、多选操作和 AI 操作员的一次复合编辑都应优先使用 `apply_batch`

### 10.5 Backend 规则

- `ensure_backend` 只承诺“保证可用”，不承诺使用本地进程还是远端服务
- `reconnect_backend` 是显式恢复，不等价于清空图或取消执行
- Backend 状态变化通过 `BackendStatusChanged` 暴露；前端不应推测内部启动阶段数

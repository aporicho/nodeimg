# 开发工作流程

```
Issue → 分支 → 设计/开发/验证 → 推送 + PR → Code Review → 合并 + 清理
```

---

## Step 1：创建 Issue

每个功能/修复对应一个 GitHub Issue。描述要做什么、为什么做。

## Step 2：创建分支

从 main 创建功能分支。根据任务规模选择方式：

**简单任务（单文件、小改动）：** 直接创建分支

```bash
git checkout -b feature/xxx
```

**复杂任务（多文件、需要并行开发）：** 使用 worktree 隔离

```bash
git worktree add -b feature/xxx .claude/worktrees/xxx
cd .claude/worktrees/xxx
```

分支命名规范：

| 前缀 | 用途 |
|------|------|
| `feature/描述` | 新功能 |
| `fix/描述` | Bug 修复 |
| `hotfix/描述` | 紧急修复 |
| `refactor/描述` | 代码重构 |
| `docs/描述` | 文档更新 |

## Step 3：设计 → 开发 → 验证

根据任务复杂度分两条路径：

### 复杂任务（新功能、架构变更、多文件改动）

使用 `superpowers:brainstorming` skill，skill 链会依次完成：

1. 设计讨论（brainstorming）
2. 实现计划（writing-plans）
3. 开发提交（executing-plans）
4. 验证（verification-before-completion）

完成后进入 Step 4。

### 简单任务（单行修复、文档更新等）

跳过 skill，直接开发并提交。推送前完成验证检查。

### 项目约定（两条路径均适用）

**Commit 信息格式：** `前缀: 描述 #Issue编号`

| 前缀 | 含义 |
|------|------|
| `feat:` | 新功能 |
| `fix:` | Bug 修复 |
| `docs:` | 文档更新 |
| `style:` | 格式调整（不影响逻辑） |
| `refactor:` | 代码重构 |
| `test:` | 添加测试 |
| `chore:` | 构建/工具等杂项 |

**验证检查清单：**

1. `cargo test --workspace` — 测试通过
2. `cargo run -p nodeimg-app --release` — 实际运行
3. 手动操作新增/修改的功能
4. 边界情况检查

## Step 4：推送 + PR

```bash
git push origin feature/xxx
gh pr create --title "feat: 描述" --body "Closes #Issue编号"
```

PR body 中用 `Closes #N` 关联 Issue，合并后自动关闭。

> main 是保护分支，不能直接 push，必须通过 PR 合并。

## Step 5：Code Review

| PR 类型 | 审查方式 |
|---------|---------|
| 简单（bug 修复、文档、小改动） | 快速扫描，检查明显问题 |
| 复杂（新功能、架构变更、多文件改动） | Skill: `superpowers:requesting-code-review` |

有问题 → 修改后 push，PR 自动更新。通过 → 进入 Step 6。

## Step 6：合并 + 清理

**普通分支：**

```bash
gh pr merge --squash                               # 合并 PR
git checkout main                                   # 切回主分支
git pull --rebase                                   # 同步 main
git branch -D feature/xxx                           # 删本地分支
```

**使用了 worktree 的分支：**

> `gh pr merge` 必须在主仓库目录执行，不能在 worktree 中执行。

```bash
cd /path/to/repo                                    # 回到主仓库目录
gh pr merge --squash                               # 合并 PR（不加 --delete-branch）
git worktree remove .claude/worktrees/xxx          # 先删 worktree
git branch -D feature/xxx                           # 再删本地分支
git checkout main                                   # 切回主分支
git pull --rebase                                   # 同步 main
```

> **注意事项：**
> - 必须先删 worktree 再删分支，worktree 存在时无法删除分支
> - `gh pr merge` 不要加 `--delete-branch`，它会在合并时尝试删本地分支，worktree 还在就会报错
> - 删分支必须用 `-D`（大写），不能用 `-d`。Squash merge 产生的新 commit 与原分支 commit 无父子关系，git 检测不到"已合并"，`-d` 会拒绝删除

---

## 关键原则

- **每个功能/修复 = 一个独立分支 + 一个 PR**，互不干扰
- **所有 commit 必须在功能分支上**，绝不在 main 上直接 commit（包括设计文档等辅助文件）
- **main 始终是经过审查的干净代码**
- **用 `git pull --rebase` 同步 main**，避免产生多余的 merge commit
- **长期分支定期 rebase main**，避免冲突积累
- **多任务并行用 worktree**，同一分支不能被两个 worktree 同时检出，完成后及时清理

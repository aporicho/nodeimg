# 主题系统

> 全局样式管理，提供颜色 token 供所有视觉模块使用。

## 模式

支持亮色 / 暗色两种模式，可在工具栏切换。所有颜色均通过 token 引用，切换模式时统一替换，无需各模块单独处理。

## 颜色 Token

### 基础 UI

| Token | 说明 |
|-------|------|
| `bg.primary` | 主背景色 |
| `bg.secondary` | 次级背景（面板、侧边栏） |
| `bg.overlay` | 弹窗 / overlay 背景 |
| `text.primary` | 主文字色 |
| `text.secondary` | 次级文字（标签、提示） |
| `text.disabled` | 禁用状态文字 |
| `border` | 通用边框色 |
| `accent` | 强调色（选中、高亮、按钮） |

### 节点分类配色

每个节点分类对应一个颜色，用于节点 header 背景和节点库分类标签。

| 分类 | 示例颜色 |
|------|----------|
| 输入 / 输出 | 蓝色系 |
| 颜色调整 | 橙色系 |
| 滤镜 | 紫色系 |
| AI 生成 | 绿色系 |
| 合成 | 红色系 |

### 引脚数据类型配色

连线和引脚圆点按数据类型着色，便于识别类型兼容性。

| 数据类型 | 颜色 |
|----------|------|
| Image | 蓝色 |
| Float | 绿色 |
| Int | 青色 |
| String | 黄色 |
| Bool | 红色 |

## 使用方式

所有视觉模块从主题系统读取 token，不硬编码颜色值。节点画布渲染时通过 `theme.node_category_color(category)` 和 `theme.pin_type_color(data_type)` 查询颜色。

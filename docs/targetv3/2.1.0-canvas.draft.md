# 节点画布

> 节点图的可视化编辑区，GUI 最核心的模块。自建 Canvas widget，不依赖第三方节点图库。

## 渲染结构

节点画布是 iced 的 `Canvas` widget，内部通过自定义 `Program` 实现全部绘制逻辑。渲染时直接从引擎读取节点图数据（读路径，不经控制器）。

```
Canvas（iced widget）
├── 节点渲染器     — 每个节点的 header / body / 引脚
│   └── 参数控件系统  — body 内嵌，渲染各类参数输入控件
└── 连线渲染器     — 已连接的线 + 正在绘制的临时线
```

## 节点结构

每个节点分三区渲染：

```
┌─────────────────────┐  ← header：节点名 + 分类配色背景
│  亮度 / 对比度       │
├──────────────────────┤  ← body：参数控件列表
│ ● 亮度    [-1  1] 0 │       引脚圆点在左/右边缘
│ ● 对比度  [-1  1] 0 │
├──────────────────────┤
│ ○ 输入图像           │  ← 输入引脚（左侧）
│              输出 ○ │  ← 输出引脚（右侧）
└─────────────────────┘
```

- **Header**：节点名称，背景色取自主题系统节点分类配色
- **Body**：参数控件系统渲染各参数（滑块、下拉框、开关等），引脚圆点紧靠左/右边缘
- **引脚**：圆点颜色取自主题系统数据类型配色，悬停时高亮显示兼容性

## 连线渲染

- 已连接的线：贝塞尔曲线，颜色跟随源引脚数据类型
- 正在绘制的临时线：从起点引脚跟随鼠标，松开鼠标时由画布控制器决定是否成立连接

## 视口

画布支持无限平移和缩放。视口状态（位置、缩放比）由画布控制器维护，渲染时作为变换矩阵传入。

## 事件处理

画布接收 iced 投递的原始鼠标/键盘事件，全部转发给画布控制器处理。画布自身不做 hit-testing，也不维护交互状态——这些都在画布控制器里。

## 执行状态显示

节点在执行期间显示视觉反馈：

| 状态 | 显示 |
|------|------|
| 等待执行 | 无变化 |
| 执行中 | header 显示进度条或旋转指示器 |
| 执行完成 | 恢复正常 |
| 执行出错 | header 变红，显示错误图标 |

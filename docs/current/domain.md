# 领域概念

定义节点系统的核心概念和规则。这些概念独立于具体实现，是整个系统的基础。

---

## 1. 节点（Node）

节点是图中的最小处理单元。接收输入，执行处理，产出输出。

### 组成结构

每个节点实例由以下部分组成（从上到下）：

1. **头部** — 标题 + 所属分类标识
2. **输入引脚区** — 0~N 个输入引脚（接收上游数据）
3. **主体** — 参数控件 + 预览区（由参数列表自动生成）
4. **输出引脚区** — 0~N 个输出引脚（传递给下游）

### 视觉结构

节点由 5 个结构组件组成：

| 结构组件 | 说明 |
|---------|------|
| NodeFrame | 节点外壳 — 背景色、圆角、阴影 |
| Header | 标题栏 — 标题文字 + 分类色标 |
| PinRow | 引脚行 — 引脚圆点（颜色由数据类型决定） + 标签，一行一个 |
| ParamRow | 参数行 — 标签 + 控件（或连线时的禁用态），一行一个 |
| PreviewArea | 预览区 — 由节点定义声明是否启用。用于显示图像缩略图、直方图等视觉内容 |

### 关键规则

- 每个节点实例有唯一 ID
- 节点不持有处理结果，结果由缓存系统管理
- 节点定义是声明式的（标题、引脚、参数、处理函数），不包含 UI 逻辑

---

## 2. 数据类型（DataType）

数据类型描述引脚传输的数据种类，决定引脚颜色、连接兼容性和类型转换。

### 内置类型

| 类型 | 含义 | 引脚颜色 |
|------|------|---------|
| Image | RGBA 图像 | 蓝色 |
| Mask | 单通道灰度图 | 绿色 |
| Float | 浮点数 | 橙色 |
| Int | 整数 | 青色 |
| Color | RGBA 颜色值 | 紫色 |
| Boolean | 布尔值 | 红色 |
| String | 文本字符串 | 灰色 |

### AI 节点的数据类型

AI 节点（如 SDXL pipeline）的中间数据类型（Model、Clip、Vae、Conditioning、Latent）不存在于前端。前端只知道这些引脚的"类型标签"用于连接兼容性检查，实际数据全部在 Python 后端内存中流转。

前端触发执行时，将 AI 节点子图序列化��� JSON 发送给后端，后端执行后返回最终结果（图像）。详见 `protocol.md`。

### 兼容规则

- 相同类型：直接连接
- 兼容类型：自动转换（如 Mask → Image，Int → Float）
- 不兼容类型：禁止连接，UI 上不允许拖线

### 兼容矩阵（内置）

| 源 → 目标 | 转换方式 |
|-----------|---------|
| Mask → Image | 灰度复制到 RGB，Alpha 设为 255（**未实现，当前为直通占位**） |
| Image → Mask | 取亮度通道（**未实现，当前为直通占位**） |
| Int → Float | 直接转换 |
| Float → Int | 四舍五入 |
| Boolean → Int | true=1, false=0 |
| Int → Boolean | 非零为 true |
| Boolean → Float | true=1.0, false=0.0 |
| Float → Boolean | 非零为 true |

### 扩展性

新类型通过注册表注册，同时声明与已有类型的兼容规则。

---

## 3. 引脚（Pin）

引脚是节点上的连接端口，数据通过引脚在节点间流动。

### 分类

| 属性 | 输入引脚 | 输出引脚 | 参数引脚 |
|------|---------|---------|---------|
| 方向 | 接收数据 | 发送数据 | 接收数据（可选） |
| 连接数 | 最多 1 条 | 不限 | 最多 1 条 |
| 位置 | 节点左侧 | 节点右侧 | 节点左侧（参数控件旁） |

### 参数引脚的特殊性（v2，未实现）

- 每个参数自动生成一个同类型的输入引脚
- 未连线时：显示手动控件（滑块、下拉框等），用户直接操作
- 已连线时：控件禁用，值由上游节点提供
- 这使任何参数都可以被上游节点动态驱动

### 引脚属性

- 名称（显示标签）
- 数据类型（决定颜色和兼容性）
- 必需/可选（必需引脚未连线时节点显示提示）

---

## 4. 连接（Connection）

连接是两个引脚之间的数据通道，从输出引脚指向输入引脚。

### 规则

- 只能从输出引脚连向输入引脚（或参数引脚）
- 一个输入引脚最多接受 1 条连接（新连线替换旧连线）
- 一个输出引脚可以连出任意多条连接
- 数据类型必须兼容（同类型或存在转换规则）
- 禁止形成环路（图必须是 DAG）

### 连接时的行为

1. 用户拖线时，UI 只高亮兼容的目标引脚
2. 连接建立后，如果需要类型转换，求值时自动插入转换
3. 连接建立/断开时，触发下游节点缓存失效

---

## 5. 约束（Constraint）

约束描述参数值的合法范围，决定自动生成的控件类型。

### 内置约束

| 约束 | 含义 | 参数 |
|------|------|------|
| Range(min, max) | 数值范围 | 最小值、最大值 |
| Enum(options) | 枚举选项 | 选项列表（标签 + 值） |
| FilePath(filters) | 文件路径 | 文件类型过滤（如 `["png", "jpg"]`） |

### 约束与控件的映射

控件由「数据类型 + 约束」组合决定。一个组合可以对应多个可用控件，系统自动选择默认控件：

| 数据类型 + 约束 | 可用控件 | 默认 |
|----------------|---------|------|
| Float + Range | 滑块、数值输入框 | 滑块 |
| Int + Range | 整数滑块、数值输入框 | 整数滑块 |
| Int + Range(0,1) | 整数滑块、复选框 | 整数滑块 |
| Boolean + 无约束 | 复选框、下拉框(是/否) | 复选框 |
| Color + 无约束 | 颜色选择器 | 颜色选择器 |
| 任意类型 + Enum | 下拉框、单选按钮组 | 下拉框 |
| 任意类型 + FilePath | 文件选择按钮 | 文件选择按钮 |

### 无约束的情况

参数可以没有约束，此时根据数据类型提供默认控件（如 Float 无约束 → 数值输入框）。

### 扩展性

新约束通过注册表注册，同时注册对应的控件映射。

---

## 6. 参数（Parameter）

参数是节点内部的可调值，由用户手动设置或由上游连线驱动。

### 参数的完整声明

- 名称（显示标签）
- 数据类型（Float / Int / Boolean / Color 等）
- 约束（Range / Enum / FilePath / 无）
- 默认值
- 控件覆盖（可选）— 不指定则用默认控件，指定则使用指定的兼容控件

### 行为规则

- 参数值变化时，触发当前节点及下游缓存失效
- 参数自动生成一个同类型的参数引脚（见引脚章节）
- 控件自动由「数据类型 + 约束」决定（见约束章节）
- 序列化时，参数名 + 值一起保存；反序列化时缺失字段用默认值填充

### 示例

Color Adjustment 节点的参数声明：

| 参数名 | 数据类型 | 约束 | 默认值 | 控件覆盖 |
|-------|---------|------|-------|---------|
| brightness | Float | Range(-1.0, 1.0) | 0.0 | — |
| saturation | Float | Range(-1.0, 1.0) | 0.0 | — |
| contrast | Float | Range(-1.0, 1.0) | 0.0 | — |

## 7. 图编辑（Graph Editing）

用户通过以下操作编辑节点图：

- **创建节点** — 通过菜单选择节点类型，在画布上创建实例
- **删除节点** — 移除节点及其所有连接，触发下游失效
- **连接引脚** — 从输出引脚拖线到输入引脚，建立数据通道
- **断开引脚** — 移除连接，触发下游失效
- **菜单** — 节点创建入口，按分类展示所有可用节点，支持搜索过滤

---

## 8. 求值（Evaluation）

求值是将节点图转化为计算结果的过程。

### 求值策略

- **按需求值：** 从目标节点（如 Preview、Save Image）出发，递归向上游求值
- **拓扑顺序：** 保证上游节点先于下游计算，不重复计算
- **可并行：** 拓扑排序中，无依赖关系的节点之间没有执行顺序约束，可并行求值

### 求值流程

1. 从目标节点出发，收集所有上游依赖
2. 拓扑排序，检测环路（环路 → 报错）
3. 按序求值每个节点：
   - 收集输入：已连线 → 取上游输出；未连线 → 取参数手动值
   - 类型转换：输入类型与期望不符时，调用转换函数
   - 执行处理函数，产出结果
   - 缓存结果

### 缓存失效

- 参数变化 → 当前节点 + 所有下游节点失效
- 连接变化（建立/断开） → 受影响节点 + 下游失效
- 只重算失效的节点，未失效的直接读缓存

### 错误处理

- 必需输入未连线 → 节点显示提示，不参与求值
- 处理函数异常 → 当前节点标记错误，下游节点收到空输入
- 错误不扩散：一个节点出错不影响图中其他独立路径

## 9. 计算后端（Compute Backend）

计算后端是执行 AI 推理的独立 Python HTTP 服务。

### 执行模型

前端负责节点图的编辑。触发执行时：
1. 前端将 AI 节点子图序列化为 JSON
2. 发送**一次** HTTP 请求给后端
3. 后端接收完整的图，自行拓扑排序，逐节点执行
4. 所有中间数据（模型、张量）留在 Python 内存中
5. 后端返回最终输出（图像）

前端不参与 AI 节点的执行过程。详见 `protocol.md`、`backend-domain.md`、`backend-architecture.md`。

### 动态节点发现

前端启动时调用 `GET /node_types` 获取后端支持的节点类型列表。后端返回每个节点的引脚和参数定义，前端据此动态注册 AI 节点到 NodeRegistry，自动生成 UI。后端新增节点类型时，前端无需修改代码。

### 本地与远程

- **本地**：后端跑在 localhost:8188
- **远程**：修改 BackendClient 的 URL 即可
- 前端代码不需要修改
